# Riot Stats API

**Бэкенд-сервис для получения, хранения и анализа игровой статистики игроков Riot Games (League of Legends).**

---

## 🔑 Доступ к Riot API

Проект использует официальный API от Riot Games.

Для запуска приложения необходимо получить `Riot Developer API Key`:

1. Зарегистрироваться на сайте `developer.riotgames.com`
2. Создать бесплатный `Developer Key`
3. Добавить ключ в переменные окружения или `.env` файл

> ⚠️ `Developer Key` действует только 24 часа, после чего его необходимо сгенерировать заново.

### Лимиты Developer Key

* `20` запросов в секунду
* `100` запросов за `2 минуты` на один хост

### Тестовый аккаунт

Для тестирования можно использовать следующий аккаунт:

* **Игрок:** `G2 SkewMond#3327`
* **Регион:** `EUW`

## 🚀 Запуск проекта

### 1. Настройка конфигурации

Скопируйте файл конфигурации и укажите ваш Riot API ключ:

```bash
cp .env.example .env
```

Затем откройте `.env` и заполните:

```env
RIOT_API_KEY=RGAPI-ваш_ключ_здесь
```

---

### 2. Запуск проекта

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу:

* **Swagger UI:** `http://localhost:8000/docs`
* **OpenAPI schema:** `http://localhost:8000/openapi.json`

## 🛠 Архитектура и стратегия работы

* Загружаются **последние 20 матчей** в режиме **SoloQ** (`queue_id = 420`)
* При синхронизации сохраняются только **новые матчи** (`deduplication` по `match_id`)
* Основной источник данных — локальная база **PostgreSQL**
* Запросы к Riot API минимизируются за счёт кеширования данных в БД

### Обработка ошибок

* При `429 Too Many Requests` учитывается заголовок `Retry-After`
* При ошибках `5xx` выполняются автоматические повторные попытки с экспоненциальной задержкой (`exponential backoff`)

## 📡 API Эндпоинты

| Метод  | Endpoint                                    | Описание                    |
| ------ | ------------------------------------------- | --------------------------- |
| `POST` | `/players/{game_name}/{tag_line}/sync`      | Синхронизация данных игрока |
| `GET`  | `/players/{game_name}/{tag_line}`           | Получение профиля и рангов  |
| `GET`  | `/players/{game_name}/{tag_line}/matches`   | Последние матчи игрока      |
| `GET`  | `/players/{game_name}/{tag_line}/champions` | Статистика по чемпионам     |
| `GET`  | `/healthz`                                  | Проверка состояния сервиса  |
