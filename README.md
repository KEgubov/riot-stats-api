# Riot Stats API

**Бэкенд-сервис для получения, хранения и анализа игровой статистики игроков Riot Games (League of Legends).**

## 🚀 Запуск

### 1. Настройка конфигурации

Скопируйте файл конфигурации и укажите ваш Riot API ключ:

```bash
cp .env.example .env
```

Затем откройте `.env` и заполните:

```env
RIOT_API_KEY=RGAPI-ваш_ключ_здесь
```

### 2. Запуск проекта

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу:

**Интерактивная документация (Swagger):** [http://localhost:8000/docs](http://localhost:8000/docs)

---

## 🛠 Стратегия работы

- **Глубина матчей**: Сервис загружает **последние 20 матчей** в режиме **SoloQ** (queue ID = 420).
- **Синхронизация**: При вызове эндпоинта синхронизации сервис получает список ID матчей и сохраняет в базу **только новые** (дедупликация по `match_id`).
- **Хранение данных**: Все данные матчей берутся из локальной базы **PostgreSQL** — минимизируются запросы к Riot API.
- **Обработка ошибок**:
  - При **429 Too Many Requests** учитывается заголовок `Retry-After`.
  - При ошибках **5xx** — автоматические повторные попытки с экспоненциальной задержкой.

## 📡 Эндпоинты

| Метод | Путь                                      | Описание                                      |
|-------|-------------------------------------------|-----------------------------------------------|
| `POST` | `/players/{game_name}/{tag_line}/sync`    | Запуск синхронизации данных игрока            |
| `GET`  | `/players/{game_name}/{tag_line}`         | Получение профиля и текущих рангов            |
| `GET`  | `/players/{game_name}/{tag_line}/matches` | Список последних матчей из базы               |
| `GET`  | `/players/{game_name}/{tag_line}/champions` | Агрегированная статистика по чемпионам     |
| `GET`  | `/healthz`                                | Проверка здоровья сервиса                     |
